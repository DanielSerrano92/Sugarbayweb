13) Incidencia adicional checkout (direccion incompleta)
- Error reportado:
  - En checkout aparecia "Esta direccion no esta completa para checkout" pese a tener direcciones guardadas.
- Causa real:
  - El selector de checkout priorizaba direccion predeterminada/reciente sin priorizar primero que fuera completa.
  - Si una direccion predeterminada tenia solo un nombre en recipientName (sin apellido), al mapear a firstName/lastName quedaba lastName vacio y se marcaba incompleta.
  - En ese escenario, aunque existia otra direccion completa guardada, no se priorizaba.

14) Correccion aplicada para checkout
- Archivo modificado:
  - components/checkout/checkout-flow.tsx
- Cambios:
  - Se agrego priorizacion de direcciones guardadas completas antes que incompletas.
  - Orden de prioridad aplicado: completa > predeterminada > mas reciente.
  - El auto-seleccionado inicial de shipping/billing ahora solo selecciona direcciones completas.
- Archivo modificado:
  - lib/repositories/checkout.ts
- Cambios:
  - Se aplico la misma priorizacion en getCheckoutPrefill para mantener coherencia server/client.
  - Se incorporo validacion local de completitud de direccion para el ranking.

15) Como validar esta correccion de checkout
- Inicia sesion y entra en /checkout.
- Verifica que:
  - Si existe al menos una direccion completa guardada, se prioriza esa automaticamente.
  - No se selecciona por defecto una direccion incompleta cuando hay otra completa disponible.
  - Puedes continuar al pago cuando shipping/billing cumplen validacion.
- Verificacion tecnica ejecutada:
  - Con datos reales actuales, el selector priorizado escoge:
    id cmpa0kr6w0001jci84giyo3lj (recipientName: "Daniel Solis")
    por delante de la direccion incompleta con recipientName "Miguel".

16) Incidencia adicional checkout (campo opcional bloqueaba direccion)
- Error reportado:
  - Si no se rellenaba un campo considerado opcional por el usuario (telefono), la direccion guardada quedaba bloqueada en checkout.
- Causa real:
  - En checkout el telefono se validaba como obligatorio y ademas se usaba como criterio de "direccion completa".
  - En cuenta, el telefono se permite vacio (nullable), por lo que habia inconsistencia entre formularios.
