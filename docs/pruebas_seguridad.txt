Pruebas seguridad - error en inputs de cambiar contrasena
Fecha: 2026-05-17

1. Objetivo
Corregir el error que se producia al escribir en los campos de "Cambiar contrasena" dentro de Mi cuenta > Seguridad.

2. Error reportado
Tipo:
- `Console TypeError`

Mensaje:
- `Cannot read properties of null (reading 'value')`

Ubicacion observada:
- `components/account/account-dashboard.tsx`
- En los `onChange` de:
  - contrasena actual
  - nueva contrasena
  - repetir nueva contrasena

3. Causa tecnica
En los handlers se estaba usando `event.currentTarget.value` dentro del callback funcional de `setPasswordForm`.
Cuando React libera/reutiliza el evento sintetico, `currentTarget` puede quedar en `null`, y al evaluarse despues dentro del updater aparece el error.

4. Solucion aplicada
Se capturo el valor del input antes de llamar al `setState`:
- `const value = event.target.value;`
- Luego se usa `value` dentro del updater en lugar de leer del evento.

5. Archivos modificados
- `components/account/account-dashboard.tsx`

6. Cambios concretos
Se actualizaron 3 handlers `onChange` en la seccion Seguridad:
- `currentPassword`
- `newPassword`
- `confirmPassword`

Patron anterior:
- `currentPassword: event.currentTarget.value`

Patron nuevo:
- `const value = event.target.value`
- `currentPassword: value`
