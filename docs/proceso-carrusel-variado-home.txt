1. Objetivo de la tarea
Corregir el carrusel variado de la Home (debajo del carrusel de videos) para que cada card navegue segun su tipo:
- Producto: imagen/titulo al detalle de producto y boton a Tienda.
- Noticia: imagen/titulo a Noticias con esa noticia expandida y boton a Noticias.
- Concierto: imagen/titulo a Proximos conciertos con modal abierto del concierto y boton a Conciertos.
Sin cambiar el diseno visual del carrusel.

2. Estado inicial del carrusel variado
- El carrusel usaba un unico `href` y un unico `ctaLabel` por slide.
- Imagen y titulo no eran clicables.
- Solo el boton inferior era clicable.
- Noticias no leia query params para expandir una noticia concreta.
- Conciertos no leia query params para abrir el modal de un concierto concreto.

   - Noticia:

6. Logica final de navegacion por tipo de item
- Producto (`store`)
  - Imagen/titulo -> `/store/<slug>`
  - Boton -> `/store`
  - Texto boton -> `Tienda`
- Noticia (`news`)
  - Imagen/titulo -> `/band/news?news=<slug>`
  - Boton -> `/band/news`
  - Texto boton -> `Noticias`
- Concierto (`concert`)
  - Imagen/titulo -> `/concerts/upcoming?concert=<slug>`
  - Boton -> `/concerts/upcoming`
  - Texto boton -> `Conciertos`

7. Explicacion de como se abre una noticia especifica expandida
- Se usa query param `news`.
- En la pagina de noticias se detecta ese valor.
- En el componente cliente del listado:
  - Se busca el item por `slug`.
  - Si existe, arranca expandido.
  - Se hace scroll al card con `id="news-<slug>"`.
- Si el slug no existe, la pagina funciona normal sin errores.

8. Explicacion de como se abre el modal de un concierto especifico
- Se usa query param `concert`.
- En la pagina de proximos conciertos se detecta ese valor.
- En el componente cliente de cards:
  - Se busca el concierto por `slug`.
  - Si existe, se inicializa `selectedConcertId` con ese concierto.
  - El modal se abre automaticamente porque `selectedConcert` ya tiene valor.
- Si el slug no existe, la pagina carga normal sin modal y sin errores.

9. Explicacion de como se navega al detalle de producto
- El carrusel usa el `slug` del producto destacado para construir `mainHref`:
  - `/store/<slug>`
- Imagen y titulo del slide de producto comparten ese destino.
- El boton inferior no va al detalle: va a `/store` con texto `Tienda`.

10. Errores encontrados y solucion aplicada
- Error de lint `react-hooks/set-state-in-effect`:
  - Aparecio en:
    - `components/band/news-list-client.tsx`
    - `components/concerts/concert-cards-client.tsx`
    - `components/home/home-header-carousel.tsx` (logica previa existente)
  - Solucion:
    - Se elimino `setState` sincrono dentro de efectos.
    - Noticias y conciertos ahora inicializan estado desde props al montar.
    - Carrusel Home ahora normaliza indice activo en lugar de resetear con efecto.
- Build fallido por descarga de fuente Google (`Press Start 2P`):
  - Causa: fallo de conexion al pedir `https://fonts.googleapis.com/...`.
  - No es un error de logica de este cambio; afecta build en este entorno sin acceso correcto a esa fuente.
