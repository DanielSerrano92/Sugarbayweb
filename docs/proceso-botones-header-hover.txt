Proceso de ajuste de hover en botones del header y eliminacion de contorno interno
Fecha: 2026-05-17

1. Objetivo de la tarea
Ajustar el hover de los botones principales del header para que el fondo no cambie, mantener el hover de tamano actual y eliminar el contorno interno del header que generaba una linea visual sobre el submenu.

2. Estado inicial de los botones del header
- Los botones principales (`.sb-header-tab`) cambiaban fondo en hover (`background: #ecc2f5`).
- El hover aplicaba efecto de tamano con `transform: translateY(-2px) scale(var(--sb-header-hover-scale))`.
- El texto en hover no se pintaba en rosa neon.

3. Problema detectado
- El hover necesitaba mantener el background fijo (sin cambio de fondo).
- Se debia conservar el hover de tamano existente.
- Aparecia una linea visual que cortaba letras en submenu, atribuida al contorno interno del header.

4. Archivos revisados
- `components/layout/header-client.tsx`
- `app/globals.css`

5. Archivos modificados
- `app/globals.css`
- `docs/proceso-botones-header-hover.txt`

6. Cambios realizados paso a paso
1. Se reviso el bloque de estilos de `.sb-header-tab` para hover/transiciones.
2. Se elimino el cambio de fondo en hover del boton principal, dejando el mismo color base.
3. Se aplico color rosa al texto en hover.
4. Se anadio glow/sombra neon rosa al texto en hover.
5. Se mantuvo intacto el `transform` de hover para conservar el mismo efecto de tamano.
6. Se localizo el contorno interno del header en `.sb-header-shell::after`.
7. Se elimino exclusivamente ese contorno interno dejando solo `content: none`.

7. Explicacion de como se dejo fijo el background en hover
En `.sb-header-tab:hover` y `.sb-header-nav-item:hover > .sb-header-tab` se cambio `background` a `#d8d8d8`, igual que el estado normal del boton.

8. Explicacion de como se aplico texto rosa en hover
En los mismos estados hover de boton principal se definio `color: #ff4fcf !important`.

9. Explicacion de como se aplico sombra/glow neon rosa al texto
Se anadio `text-shadow` en hover:
- `0 0 7px rgb(255 79 207 / 52%)`
- `0 0 12px rgb(255 79 207 / 32%)`
Ademas se extendio la transicion para `color` y `text-shadow`.

10. Confirmacion de que se mantuvo el hover de tamano de botones
Confirmado: no se modifico `transform: translateY(-2px) scale(var(--sb-header-hover-scale))`.

11. Confirmacion de que no se cambio el hover del submenu
Confirmado: no se tocaron reglas de `.sb-header-dropdown-item:hover` ni `.sb-header-dropdown-item-active`.

12. Causa del contorno interno del header
El contorno interno provenia del pseudo-elemento `.sb-header-shell::after`, que dibujaba un borde interno animado dentro del contenedor del header.

13. Explicacion de como se elimino el contorno interno
Se cambio el bloque `.sb-header-shell::after` para que no renderice el contorno interno: `content: none`.

14. Confirmacion de que se mantuvo el borde exterior del header
Confirmado: no se modifico el borde exterior de `.sb-header-shell` (`border: 6px solid #fff`).

15. Confirmacion de que se mantuvo el background interior con animacion
Confirmado: no se modificaron `background`, `background-size` ni `animation: sb-header-shell-neon` del contenedor principal del header.

16. Confirmacion de que no se cambio la logica de navegacion
No se modificaron rutas, estructura de links, logica de apertura/cierre de submenu, ni codigo TS/JS del header.

17. Pruebas manuales realizadas
- Revision de CSS aplicada a botones principales: fondo fijo en hover, texto rosa y glow neon.
- Verificacion de que el hover de tamano se mantiene con el mismo `transform` previo.
- Verificacion de que el hover del submenu no fue modificado a nivel de reglas CSS.
