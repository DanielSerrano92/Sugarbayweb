Correccion de direcciones guardadas en checkout
Fecha: 2026-05-17

1. Objetivo de la tarea
Integrar correctamente las direcciones guardadas de usuario en checkout para que puedan:
- seleccionarse como envio/facturacion,
- convivir con entrada manual,
- respetar direccion predeterminada,
- mantener el flujo actual de pago sin romper carrito, Stripe ni autenticacion.

2. Error encontrado
- Las direcciones se actualizaban correctamente en base de datos.
- Las direcciones se mostraban correctamente en "Mis direcciones".
- En checkout no habia selector de direcciones guardadas y no se podia elegir entre varias.
- El checkout solo usaba un prefill inicial y no permitia seleccionar/alternar direcciones guardadas.

3. Causa tecnica del problema
- `components/checkout/checkout-flow.tsx` no cargaba ni mostraba listado de direcciones guardadas.
- Solo existian formularios manuales con `initialShipping`/`initialBilling`.
- No habia estado de "direccion guardada seleccionada" vs "direccion manual".
- No habia estados de carga/error para direcciones en checkout ni fallback claro de seleccion.

7. Como se cargan ahora las direcciones guardadas en checkout
- Checkout carga direcciones mediante `GET /api/account/addresses` al montar `CheckoutFlow`.
- La API filtra por usuario autenticado en servidor (`getSessionUser` + `listAccountAddresses(session.userId)`).
- Si la carga falla, se muestra error visible y se mantiene modo manual operativo.

8. Como se selecciona una direccion guardada
- Se muestra lista de direcciones por tipo:
  - envio (`SHIPPING`)
  - facturacion (`BILLING`)
- El usuario selecciona mediante radio + card resaltada.
- Al seleccionar una direccion:
  - se mapea al formulario de checkout,
  - queda en modo `saved`,
  - el formulario queda bloqueado para evitar edicion accidental.

9. Como se usa una direccion manual diferente
- Siempre existe opcion de radio:
  - "Usar otra direccion de envio"
  - "Usar otra direccion de facturacion"
- Al elegirla:
  - el modo pasa a `manual`,
  - se habilitan los campos del formulario,
  - el usuario puede introducir una direccion distinta para ese checkout.

10. Como se gestiona la direccion predeterminada
- Si hay direccion `isDefault` del tipo correspondiente, se prioriza para seleccion inicial.
- Si no hay `isDefault`, se usa la primera direccion valida de ese tipo.
- Si la direccion actual ya coincide con una guardada, se mantiene esa coincidencia como seleccion activa.

11. Como se gestiona facturacion igual/diferente a envio
- Si "usar la misma direccion de facturacion" esta activo:
  - checkout usa direccion de envio para facturacion (como antes).
- Si esta desactivado:
  - se habilita selector y formulario de facturacion independiente.
  - puede ser guardada o manual.

12. Como se valida que la direccion pertenece al usuario autenticado
- Las direcciones cargadas en checkout provienen de `/api/account/addresses`.
- Esa API usa sesion servidor y consulta por `session.userId`.
- No se acepta direccion de otro usuario porque no se expone en el listado.
- En checkout no se confia en `addressId` para crear pedido: se envia snapshot validado del formulario.

13. Como se guarda la direccion final en el pedido
- `POST /api/checkout` sigue validando `shipping/billing` con `checkoutPayloadSchema`.
- `saveCheckoutAddresses` actualiza direcciones por defecto del usuario segun snapshot final.
- `createPendingOrderFromCart` crea pedido con snapshot de direccion en campos `shipping*`/`billing*`.
- El pedido conserva la direccion final efectiva usada en checkout.
