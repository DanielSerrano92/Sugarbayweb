Hero sobre background y degradado real

Proyecto: Sugarbayweb. Fecha: 2026-05-02. Archivos principales: components/ui/page-shell.tsx y app/globals.css.

Resultado actual: cada pagina renderizada con PageShell tiene una unica seccion principal con imageBackground. Dentro de esa misma seccion se pinta primero la imagen superior del hero y despues el contenido. El footer global queda fuera de la seccion con background.

Que se hizo para dejar el hero encima del background

  1. Se elimino el componente PageHero como bloque independiente.
  2. PageShell paso a renderizar directamente una unica <section className="page-background">.
  3. La imagen que antes funcionaba como hero se coloco al inicio de esa misma seccion, dentro de .page-header-image.
  4. El contenido de la pagina se coloco debajo, dentro de .page-content-wrapper, sin salir del wrapper con background.
  5. La clase .page-background > * da position: relative y z-index: 1 a los hijos para que imagen y contenido queden visualmente por encima del patron de fondo.

Como se consiguio el degradado del hero

El degradado no usa una franja de color, ni blanco, ni negro superpuesto. Se usa una mascara CSS real sobre la imagen: mask-image y -webkit-mask-image. En una mascara, la parte negra mantiene visible el elemento y la parte transparente lo oculta. Por eso la imagen del hero se ve normal hasta el 76% de su altura y desde ahi pierde opacidad gradualmente hasta desaparecer en el borde inferior.

Como la imagen se vuelve transparente en el bottom, lo que se ve a traves es el background-image de la seccion padre .page-background. Esto funde la cabecera con el fondo real de la pagina sin simularlo con colores.

Codigo modificado: PageShell

Archivo: components/ui/page-shell.tsx 

import Image from "next/image";
import type { ReactNode } from "react";

type PageShellProps = {
  title: string;
  eyebrow?: string;
  description?: string;
  actions?: ReactNode;
  children: ReactNode;
  contentClassName?: string;
  sectionClassName?: string;
};

const DEFAULT_CONTENT_CLASS_NAME = "space-y-6";
const PAGE_HEADER_IMAGE_SRC =
  "https://ik.imagekit.io/gq1enkszp/fotos/proximos.png?tr=w-2400,h-760,cm-extract,fo-top&updatedAt=1777405652441";

export default function PageShell({
  title,
  eyebrow,
  description,
  actions,
  children,
  contentClassName,
  sectionClassName,
}: PageShellProps) {
  const sectionClasses = [
    "page-background",
    "relative",
    "flex-1",
    "w-screen",
    sectionClassName,
  ]
    .filter(Boolean)
    .join(" ");

  const contentClasses = [
    "page-content-wrapper",
    contentClassName ?? DEFAULT_CONTENT_CLASS_NAME,
  ].join(" ");

  return (
    <section
      className={sectionClasses}
      style={{
        marginLeft: "calc(50% - 50vw)",
        marginRight: "calc(50% - 50vw)",
      }}
    >
      <div className="page-header-image">
        <Image
          src={PAGE_HEADER_IMAGE_SRC}
          alt={`Cabecera ${title}`}
          width={2400}
          height={760}
          priority
          sizes="100vw"
          unoptimized
          className="page-header-img"
        />
      </div>
      <h1 className="sr-only">{title}</h1>
      {eyebrow || description || actions ? (
        <div className="sr-only">
          {eyebrow ? <p>{eyebrow}</p> : null}
          {description ? <p>{description}</p> : null}
          {actions ? <div>{actions}</div> : null}
        </div>
      ) : null}
      <div className={contentClasses}>{children}</div>
    </section>
  );
}

Codigo modificado: estilos globales

Archivo: app/globals.css 

.page-background {
  position: relative;
  isolation: isolate;
  background-color: var(--sb-bg-main);
  background-image: url("https://ik.imagekit.io/gq1enkszp/fotos/rejilla%20(7).png");
  background-position: top center;
  background-repeat: repeat;
  background-size: min(100vw, 1100px) auto;
}

.page-background > * {
  position: relative;
  z-index: 1;
}

.page-header-image {
  width: 100%;
  overflow: hidden;
}

.page-header-img {
  display: block;
  width: 100%;
  height: auto;
  object-fit: cover;
  mask-image: linear-gradient(
    to bottom,
    black 0%,
    black 76%,
    transparent 100%
  );
  mask-repeat: no-repeat;
  mask-size: 100% 100%;
  -webkit-mask-image: linear-gradient(
    to bottom,
    black 0%,
    black 76%,
    transparent 100%
  );
  -webkit-mask-repeat: no-repeat;
  -webkit-mask-size: 100% 100%;
}

.page-content-wrapper {
  width: 100%;
  max-width: 80rem;
  margin: 0 auto;
  padding: 1.5rem 1rem;
}

@media (min-width: 1024px) {
  .page-content-wrapper {
    padding: 2rem;
  }
}

Decisiones concretas

  • background-repeat: repeat mantiene el patron repetible segun lo pedido.
  • background-size: min(100vw, 1100px) auto hace que el fondo escale hacia abajo con ventanas pequenas, pero no crezca mas al superar 1100px.
  • sizes="100vw" declara que la imagen superior ocupa el ancho de la ventana.
  • width={2400} y height={760} mantienen una relacion de aspecto estable para evitar saltos de layout.
  • La transformacion ImageKit w-2400,h-760,cm-extract,fo-top genera una cabecera amplia con foco arriba y menos recorte vertical que la version inicial de h-620.

Fuentes consultadas

  • MDN, mask-image: explica que mask-image usa una imagen o gradiente como mascara y que las zonas transparentes ocultan el elemento. https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/mask-image
  • MDN, background-size: documenta como controlar el tamano de una imagen de fondo y que los fondos con transparencia dejan ver el background-color. https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/background-size
  • MDN, min(): documenta la funcion CSS usada para limitar el tamano maximo del fondo y permitir que se reduzca con el viewport. https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Values/min
  • Next.js, Image Component: referencia de src, width, height, sizes y unoptimized para imagenes remotas y responsive. https://nextjs.org/docs/app/api-reference/components/image
  • ImageKit, Resize and crop images: referencia de transformaciones w, h, cm-extract y fo. https://imagekit.io/docs/image-resize-and-crop
